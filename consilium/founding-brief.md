## Извлечённые тезисы (интерпретация — НЕ verbatim)

Ниже — структурированный разбор брифа для последующей работы консилиума.
При расхождении с дословным текстом выше — приоритет у дословного.

1. **Цель проекта.** Консилиум агентов разных вендоров прорабатывает и тестирует
   алгоритм сжатия: сначала описывается правило (как алгоритм должен работать),
   затем правила дорабатываются, перебираются гипотезы о лучших подходах.

2. **Основной принцип.** Данные представляются в виде N-мерного куба. Каждая грань
   куба не превышает некоторого числа значений (например 256 — условно, число N
   подлежит подбору).

3. **Разреженность и карта расстояний.** Данные по кубу разбросаны неравномерно, с
   пропусками. Пропуски на каждой стороне записываются не как абсолютное значение
   на грани, а как **расстояние до следующего значения**: один пропуск → +1, два →
   +2 и т.д. Пропуск не может быть больше длины грани.

4. **Сжатие карты расстояний.** Сама карта расстояний тоже сжимается — компактной
   записью таких рядов.

5. **Цель преобразования.** Все значения сдвигаются «в угол куба», чтобы запись
   одного значения занимала не 4–5 байт, а несколько бит.

6. **Формат файла.** Сначала — карта расстояний, затем данные в виде сокращённых
   битовых последовательностей; ширина следующего значения в битах известна заранее.

7. **Открытые вопросы (research).** N-мерность под вопросом: может быть строго 3D
   или 4D, может — переменное N. Нужно перебирать варианты, исследовать уже
   применявшиеся математические методы, возможно изобрести новое на консилиуме.

8. **Метод работы.** Реализовать архиватор, погонять на тестовых данных,
   итеративно «пожимать» и через несколько итераций прийти к готовому решению.
